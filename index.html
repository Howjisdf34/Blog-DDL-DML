Estado de la base

3.1 · Concepto

Qué es el lenguaje estructurado de consulta

SQL (Structured Query Language) es el lenguaje estándar para definir, manipular y consultar la información guardada en una base de datos relacional. No describe cómo buscar los datos: describe qué datos se quieren, y el gestor decide el camino.

Es declarativo

Escribes el resultado que esperas. El optimizador del gestor (MySQL, PostgreSQL, SQL Server, Oracle) elige el plan de ejecución.

Es un estándar

Normalizado por ANSI e ISO desde 1986. Cada gestor añade su dialecto, pero el núcleo se escribe casi igual en todos.

Trabaja con conjuntos

Una instrucción no actúa sobre un registro: actúa sobre todas las filas que cumplen la condición, de una sola vez.

Elementos del lenguaje

  • InstrucciónLa orden completa, terminada en punto y coma: CREATE, INSERT, SELECT.
  • CláusulaComponente de una instrucción: FROM, WHERE, ORDER BY, GROUP BY.
  • PredicadoCondición que se evalúa como verdadero, falso o desconocido y filtra las filas.
  • ExpresiónCombinación que produce un valor: promedio * 10, nombre.
  • OperadorAritméticos, de comparación, lógicos (AND, OR, NOT) y LIKE, IN, BETWEEN.
  • FunciónDe agregado (COUNT, SUM, AVG, MAX, MIN) o escalares de texto y fecha.
  • IdentificadorEl nombre de una tabla, columna, vista o índice.
  • Tipo de datoDefine qué admite cada columna: INT, VARCHAR, DATE, DECIMAL.

Sublenguajes

SQL se reparte en cinco familias

Todas comparten sintaxis, pero cada una tiene un trabajo distinto. Este tema se concentra en las dos primeras.

DDLDefinición. Crea y modifica la estructura: tablas, columnas, índices, restricciones.
DMLManipulación. Inserta, cambia y borra los datos que viven dentro de esa estructura.
DQLConsulta. SELECT: recupera información sin alterarla. Muchos autores lo incluyen dentro del DML.
DCLControl. GRANT y REVOKE: quién puede hacer qué.
TCLTransacciones. COMMIT, ROLLBACK, SAVEPOINT: confirma o deshace un bloque de cambios.

Familia 1 · Data Definition Language

DDL: el lenguaje que dibuja el envase

Actúa sobre el esquema, no sobre los registros. Sus cambios suelen confirmarse solos (auto-commit): una vez ejecutados, no hay ROLLBACK que valga en la mayoría de los gestores.

CREATE

Crea un objeto nuevo: base de datos, tabla, vista o índice.

CREATE TABLE alumnos (
  id      INT PRIMARY KEY,
  nombre  VARCHAR(50) NOT NULL,
  grupo   CHAR(4)
);

ALTER

Modifica la estructura de un objeto que ya existe: agrega, cambia o quita columnas y restricciones.

ALTER TABLE alumnos
  ADD promedio DECIMAL(4,2);

DROP

Elimina el objeto completo. Se va la estructura y, con ella, todos los datos.

DROP TABLE alumnos;

TRUNCATE

Vacía la tabla de golpe pero conserva su definición. Es más rápido que borrar fila por fila.

TRUNCATE TABLE alumnos;

RENAME

Cambia el nombre de un objeto sin tocar su contenido.

RENAME TABLE alumnos TO estudiantes;

Familia 2 · Data Manipulation Language

DML: el lenguaje que mueve el contenido

La estructura no se entera. Aquí solo cambian las filas, y cada instrucción vive dentro de una transacción: se puede confirmar con COMMIT o deshacer con ROLLBACK.

INSERT

Agrega filas nuevas a una tabla existente.

INSERT INTO alumnos (id, nombre, grupo)
VALUES (1, 'Ana Rivas', '5B');

UPDATE

Cambia valores de las filas que cumplen la condición. Sin WHERE, cambia todas.

UPDATE alumnos
SET promedio = 8.5
WHERE id = 2;

DELETE

Borra filas concretas. La tabla sigue ahí, vacía o con lo que quedó.

DELETE FROM alumnos
WHERE promedio < 6;

SELECT

Recupera datos. Es la instrucción más usada de SQL; parte de los autores la clasifican aparte, como DQL.

SELECT nombre, promedio
FROM alumnos
WHERE grupo = '5B'
ORDER BY promedio DESC;

MERGE

Inserta o actualiza según exista o no la fila. Se conoce como upsert.

MERGE INTO alumnos a
USING nuevos n ON (a.id = n.id)
WHEN MATCHED THEN UPDATE SET a.promedio = n.promedio
WHEN NOT MATCHED THEN INSERT VALUES (n.id, n.nombre);
